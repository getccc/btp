# 🤖 跟单机器人功能指南

## 概述

跟单机器人可以自动监控目标钱包地址的交易，并实时复制执行相同的交易。支持 BSC 和 Solana 双链，提供灵活的跟单配置选项。

## ✨ 核心功能

### 1. 智能跟单
- 实时监控目标地址的 DEX 交易
- 自动识别 Swap 交易
- 根据配置自动或手动执行跟单

### 2. 灵活的金额配置
- **固定金额模式**: 每次跟单使用固定数量
- **比例模式**: 跟单目标交易金额的一定百分比
- **最大金额限制**: 设置单笔跟单的最大金额

### 3. 代币过滤
- **白名单**: 只跟单特定代币的交易
- **黑名单**: 排除特定代币的交易
- 灵活组合使用

### 4. 执行控制
- **自动执行**: 检测到交易后自动执行
- **手动确认**: 需要用户手动确认每笔跟单
- **暂停/恢复**: 随时暂停或恢复跟单

## 🎯 使用场景

### 场景1: 跟单成功的交易员
```
目标: 跟单一个盈利稳定的钱包地址
配置:
- 目标地址: 0x123...abc
- 金额模式: 固定金额 0.1 BNB
- 最大金额: 0.5 BNB
- 自动执行: 是
- 白名单: 不设置（跟单所有交易）
```

### 场景2: 谨慎跟单
```
目标: 跟单但需要手动确认每笔交易
配置:
- 目标地址: 0x456...def
- 金额模式: 比例 10%
- 最大金额: 1.0 BNB
- 自动执行: 否
- 白名单: USDT, BNB, ETH（只跟单主流代币）
```

### 场景3: 大户跟单
```
目标: 跟单大户，但按自己的资金比例
配置:
- 目标地址: 0x789...ghi
- 金额模式: 比例 1%（大户交易10 BNB，跟单0.1 BNB）
- 最大金额: 2.0 BNB
- 自动执行: 是
- 黑名单: SCAM, FAKE（排除可疑代币）
```

## 📋 配置参数详解

### 必填参数

| 参数 | 说明 | 示例 |
|------|------|------|
| 目标地址 | 要跟单的钱包地址 | 0x1234...5678 |
| 链 | BSC 或 Solana | BSC |
| 金额模式 | fixed（固定）或 percentage（比例） | fixed |

### 金额配置

**固定金额模式 (Fixed Amount)**
```
固定金额: 0.1 BNB
最大金额: 0.5 BNB

示例:
- 目标交易 5 BNB → 跟单 0.1 BNB
- 目标交易 0.05 BNB → 跟单 0.1 BNB
- 目标交易 10 BNB → 跟单 0.1 BNB（不超过最大金额）
```

**比例模式 (Percentage)**
```
比例: 10%
最大金额: 1.0 BNB

示例:
- 目标交易 1 BNB → 跟单 0.1 BNB (1 * 10%)
- 目标交易 5 BNB → 跟单 0.5 BNB (5 * 10%)
- 目标交易 15 BNB → 跟单 1.0 BNB (15 * 10% = 1.5, 但限制为最大金额 1.0)
```

### 可选参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 标签 | 配置的名称 | Copy {地址前6位}... |
| 最大金额 | 单笔跟单的最大金额 | 无限制 |
| 滑点容忍度 | 交易滑点百分比 | 1.0% |
| 自动执行 | 是否自动执行 | 是 |
| 白名单代币 | 只跟单这些代币 | 空（所有代币） |
| 黑名单代币 | 不跟单这些代币 | 空 |

## 🚀 快速开始

### 步骤 1: 连接钱包

1. 访问跟单页面
2. 选择链（BSC 或 Solana）
3. 点击 "Connect MetaMask" 或 "Connect Phantom"
4. 在钱包中确认连接

### 步骤 2: 创建跟单配置

1. 点击 "+ New Copy Trading Config" 按钮
2. 填写配置表单：
   - 输入目标地址
   - 选择金额模式
   - 设置金额参数
   - 配置可选参数
3. 点击 "Create Config"

### 步骤 3: 开始跟单

配置创建后自动开始监控。当目标地址发生交易时：

**自动执行模式**:
- 系统自动准备跟单交易
- 弹出钱包签名请求
- 确认后执行跟单

**手动确认模式**:
- 显示待确认的跟单交易
- 查看交易详情
- 点击 "Confirm" 执行或 "Cancel" 取消

### 步骤 4: 管理配置

- **暂停**: 临时停止跟单，不删除配置
- **恢复**: 继续跟单
- **停止**: 完全停止跟单
- **删除**: 删除配置，停止监控

## 🔐 安全注意事项

### 1. 目标地址选择
⚠️ **重要**: 只跟单你信任的地址
- 验证地址的历史交易记录
- 确认地址的盈利情况
- 警惕异常的交易模式

### 2. 金额控制
- 使用最大金额限制保护资金
- 从小额开始测试
- 定期检查跟单结果

### 3. 代币过滤
- 使用白名单限制跟单代币
- 黑名单排除可疑代币
- 关注新代币的风险

### 4. 自动执行风险
- 自动执行意味着无需确认即可执行交易
- 确保目标地址可靠
- 设置合理的金额限制
- 建议先使用手动确认模式测试

### 5. 余额管理
- 确保钱包有足够余额
- 预留 Gas 费用
- 定期检查余额

## ⚙️ 高级技巧

### 1. 多配置策略
```
创建多个配置，分散风险:
- 配置1: 跟单地址A，固定金额 0.1 BNB
- 配置2: 跟单地址B，固定金额 0.05 BNB
- 配置3: 跟单地址C，比例 5%，最大 0.2 BNB
```

### 2. 代币白名单策略
```
只跟单主流代币，降低风险:
白名单: BNB, USDT, USDC, ETH, BTCB

这样只会跟单这些代币的交易，忽略其他代币
```

### 3. 黑名单保护
```
排除已知的高风险代币:
黑名单: SCAM, FAKE, PONZI

这些代币的交易将被忽略
```

### 4. 动态调整
```
根据市场情况调整配置:
- 牛市: 增加跟单金额或比例
- 熊市: 减少金额，暂停部分配置
- 高波动: 使用手动确认模式
```

### 5. 监控优化
```
关注配置的统计数据:
- 成功率低(<50%): 考虑暂停或删除
- 连续失败(>3次): 检查目标地址情况
- 高成功率(>80%): 可以适当增加金额
```

## 📈 性能和限制

### 延迟
- 交易检测延迟: 2-5秒（BSC）, 1-2秒（Solana）
- 执行延迟: 取决于网络拥堵情况
- 总延迟通常在10-30秒

### 限制
- 单个用户最多100个配置
- 每个配置最多监控1个目标地址
- 历史记录保留最近200条

### Gas费用
- BSC: 约 $0.1-0.5 每笔交易
- Solana: 约 $0.001-0.005 每笔交易

## 🐛 常见问题

### Q1: 为什么跟单没有执行？
**可能原因**:
1. 配置状态不是 "active"
2. 代币在黑名单中
3. 代币不在白名单中（如设置了白名单）
4. 余额不足
5. 目标交易不是 DEX swap 交易

**解决方案**:
- 检查配置状态
- 查看代币过滤设置
- 确认钱包余额充足
- 查看跟单历史中的失败原因

### Q2: 自动执行失败怎么办？
**可能原因**:
1. 钱包拒绝签名
2. Gas 费不足
3. 滑点过大
4. 代币流动性不足

**解决方案**:
- 确保钱包已连接
- 增加 Gas 费余额
- 提高滑点容忍度
- 降低跟单金额

### Q3: 如何停止所有跟单？
**方法1**: 逐个暂停配置
- 点击每个配置的 "Pause" 按钮

**方法2**: 停止配置
- 点击 "Stop" 按钮完全停止

**方法3**: 删除配置
- 点击 "Delete" 按钮删除配置

### Q4: 跟单成功率低怎么办？
**分析原因**:
- 查看失败的交易记录
- 检查失败的错误信息
- 分析目标地址的交易模式

**改进措施**:
- 调整金额设置
- 增加滑点容忍度
- 使用手动确认模式
- 更换目标地址

### Q5: 可以同时跟单多个地址吗？
可以！创建多个配置，每个配置监控一个目标地址。

### Q6: 跟单会跟到假币/骗局吗？
可能。建议：
- 使用白名单只跟单主流代币
- 使用黑名单排除可疑代币
- 先使用手动确认模式
- 定期检查跟单结果

### Q7: 跟单延迟多少？
通常在10-30秒之间，取决于：
- 网络拥堵程度
- 区块确认速度
- 系统处理速度

### Q8: 如何选择好的目标地址？
建议标准：
- 历史盈利稳定
- 交易频率适中
- 持仓时间合理
- 避免过于频繁的交易
- 关注链上分析工具的评级

## 🔄 工作原理

### 系统架构
```
1. 地址监控
   ↓
2. 交易检测（识别 DEX Swap）
   ↓
3. 配置匹配（查找跟单此地址的配置）
   ↓
4. 过滤检查（白名单/黑名单）
   ↓
5. 金额计算（根据配置计算跟单金额）
   ↓
6. 执行准备（准备交易参数）
   ↓
7. 用户确认（自动或手动）
   ↓
8. 执行交易（通过钱包签名）
   ↓
9. 记录结果（更新统计和历史）
```

### 数据流程
```
目标地址交易
  ↓
BSC/Solana 区块链
  ↓
地址监控服务
  ↓
跟单执行器
  ↓ (WebSocket)
前端界面
  ↓ (钱包签名)
区块链执行
```
